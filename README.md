# hydrantmapper-data

Jurisdictional data packs for the [HydrantMapper](https://github.com/SethTata/HydrantMapper) iOS app. Each pack contains hydrant locations (CSV) and cross-streets data (JSON) for one municipality.

Served via GitHub Pages at: https://SethTata.github.io/hydrantmapper-data/

## Layout

- `jurisdictions/<slug>/manifest.json` — per-pack metadata + SHA256 checksums
- `jurisdictions/<slug>/intersections.json` — OSM-derived cross-streets table
- `jurisdictions/<slug>/hydrants.csv` — hydrant locations (optional; absent if pack is cross-streets-only)
- `jurisdictions.json` — top-level registry; generated by CI on every push

## Publishing a new pack

1. Generate `intersections.json` via the `add-hydrantmapper-jurisdiction` skill in the HydrantMapper repo.
2. Drop the output into `jurisdictions/<new-slug>/`.
3. Write `manifest.json` (template in `scripts/manifest_template.json`).
4. Commit and push to `main`.
5. CI validates, recomputes checksums, regenerates the registry, and deploys to `gh-pages`.

## Running scripts locally

```bash
python -m scripts.cli validate-all
python -m scripts.cli checksums
python -m scripts.cli registry
python -m scripts.cli run-all  # does all three
```

## Running tests

```bash
python -m pytest -v
```
